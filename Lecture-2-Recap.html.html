Chapter 5 · Practical Handling
safe combining

Adding and filtering data needs consistency

add rows
Use row binding when the new record has the same columns.
add columns
Use column binding or create a new variable with $.
filter rows
Use logical conditions such as age limits or category matches.
The main trap is factors: when you add new rows, the category levels must be compatible. Another subtle point: selecting one column may simplify into a vector unless you keep the data-frame shape explicitly.

Chapter 6 · Special Values
not all “missing” values mean the same thing

R distinguishes different kinds of problematic values

Inf
A result that grows beyond the largest finite number.
NaN
An undefined numeric result, such as impossible arithmetic.
NA
A missing value: something should be there, but it is unknown.
NULL
No object at all, or an absent component.
Inf = too large but still numeric
NaN = undefined numeric result
NA = missing data value
NULL = nothing is stored there
distinguish overflow, undefined, missing, and absent05
Chapter 6 · Safe Checks
defensive programming

Use the right tests

  • is.na() for missing values
  • is.nan() for undefined numeric results
  • is.infinite() for ±Inf
  • is.null() for absent objects

Rule to remember

never test missingness with
x == NA

use
is.na(x)
Comparisons involving NA often return NA, not TRUE or FALSE. That is why explicit helper functions are essential.
check deliberately, not by guessing06
Chapter 9 · Functions and Environments
where R looks for names

Function calls happen inside environments

Objects you create usually live in the global environment. Functions run in local environments. Packages contribute namespaces and a search path.
global environment
→

→
local function call

Why this matters

  • R must decide which object or function a name refers to
  • different packages may offer functions with the same name
  • overwriting names like T or F causes confusion

Chapter 9 · Argument Matching
how R matches inputs

R can match function arguments in several ways

exact
Safest and clearest. Name the argument fully.
partial
Convenient, but risky if the abbreviation is ambiguous.
positional
Works by order, so you must know the function signature.
mixed + ...
Some arguments are named, others passed by position or through ellipsis.
Best habit: prefer clear argument names for readability and fewer mistakes. Treat partial matching as a convenience, not a teaching default.
exact names are safest partial matching can fail ... passes extra options
